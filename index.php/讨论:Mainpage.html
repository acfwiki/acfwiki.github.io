关于首页对于宽屏的支持

首先表示非常喜欢Metro风格，因此一直在关注AcWiki的改版。但这次取消了宽屏支持，导致首页右侧空出了一片，对此我略有残念。诚然，应该顾及到非宽屏如1024*768的用户，但也应在此基础上考虑宽屏用户。对此，可否通过如下两种途径，解决这个问题？（或者有没有其他更好的方法？）

  • 通过代码，判断该访问者的屏幕分辨率，并通过算法实现每个栏目的宽和高。
  • 无需增加代码，将首页居中。--Zxqdx 2012年4月20日 (五) 22:42 (BNT)

补充一句：改版刚开始的Wiki首页支持宽屏，挺好的。因此，我觉得可以设置两个版本的主页，都是Metro风格，一个是现在的非宽屏版，一个是刚才提到的宽屏版，然后判断用户的屏幕分辨率来选择跳转到对应的首页版本。--Zxqdx 2012年4月20日 (五) 22:50 (BNT)

  • 抱歉我的电脑目前右边也是白白一片，ATM君要求适应1024宽度起的电脑，所以原来的版本被否决了。用代码来做这个并不难，但是要一个美观的tiles分布是很需要调整的，可能代码匹配出的结果在某种分辨率下反而不美观。目前本人事务紧张暂时将不打算继续修改主页。--Superx 2012年4月26日 (四) 06:23 (BNT)
    • 了解了~不过现在这个首页用了几天后倒也适应了。我觉得只要内容页面是支持宽屏的，用户就应该能接受。--Zxqdx 2012年4月26日 (四) 09:57 (BNT)
因为要考虑到左侧边栏，居中效果不会好。宽屏用户很少有人会把浏览器全屏的；浏览器的宽度也不一定。用代码检测匹配分辨率费时费力不讨好效率低下，既然没有绝对的必要，当前的版本我认为还是继续沿用。--by WIKI管理员ATM 2012年4月26日 (四) 15:20 (BNT)

编辑首页而标题为空导致无法编辑的问题【已解决】

经过一番尝试，我发现我不能编辑“Mainpage”页面都是被Mainpage源代码的第一句给害的： 

{{DISPLAYTITLE:<span style="display:none">{{FULLPAGENAME}}</span>}}

这句话使Mainpage的标题为空，于是乎无论对Mainpage进行什么操作（预览、保存），wiki都认为我在对一个标题为空的页面（标题为""）进行操作，因此wiki自动将空标题改为“条目名称”，继而使我无法编辑首页。

但是这句话如果去掉，就会导致首页上出现“Mainpage”这个多余的标题行。描述完毕，各位大大求解~~

--By Zxqdx（留言） 2012年7月18日 (三) 03:34 (UTC)。

OK，终于解决了。虽然我始终没弄明白为啥光驱、互撸娃都能改就我不行，但是谢天谢地总算搞定了。

我的方法是，去掉Mainpage上面的这段代码： 

{{DISPLAYTITLE:<span style="display:none">{{FULLPAGENAME}}</span>}}

而在Mediawiki:common.css里面放上这段代码： 

body.page-Mainpage h1.firstHeading { display:none; }

这样我就能正常编辑首页了，并且首页上的标题依旧被隐藏。

同时我也发现一个隐藏任意页面标题的方法：只需要在Mediawiki:common.css里面放上这段代码： 

body.page-XXXXX h1.firstHeading { display:none; }

就可以把“XXXXX”这一页面的标题隐藏了。

同理： 

body.page-AA_BBB h1.firstHeading, body.page-CCCCC h1.firstHeading  { display:none; }

就可以同时把“AA BBB”和“CCCCC”这两个页面的标题隐藏起来。

这种css隐藏标题的方法有两个好处：

  • 可以彻底隐藏标题，包括标题下方的一条分隔符（横线）；
  • 因为是在common.css里面改的，所以适用于所有wiki皮肤。
